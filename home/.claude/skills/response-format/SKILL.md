# 対応完了レポート出力ルール

ファイル編集・コマンド実行・調査などの作業を伴うユーザー依頼の最終応答時に、必ず以下のテンプレに沿って日本語のプレーンテキストMarkdownで報告する。

## 適用判断（最重要）

このスキルを適用するかは「対応の規模」で判断する。

### 適用する（必須）

- 複数ツール呼び出し（Edit/Write/Bashの実行などが3回以上）を伴った作業の最終応答
- 複数ファイル編集・新規ファイル作成を含む実装の完了
- 調査・分析の結果報告
- 複数ターンに分かれた作業の最終完了時
- 設定変更・リファクタリング・移行作業の完了

### 適用しない（スキップ）

- 単発の質問への即答（ツール呼び出しなし）
- 1〜2回のRead/Bash読み取りのみで完結する確認系応答
- ユーザーの質問への会話的な回答
- 進捗報告・中間確認（最終応答ではないため）

判断に迷う場合は適用する。

## 出力ルール

- 必ずプレーンテキストのMarkdownで出力する
- `{}`で囲われた説明文を実内容に置き換える
- 出力後、システム通知で完了を報告する（環境により省略可）

## 出力フォーマット

```markdown
# {私の入力全体を要約し、タイトルに変換}

{私の入力の全て}

## 解釈の要約

- {私の入力に対して、あなたの解釈の要約を箇条書き}

## 対応内容

- {実施した対応内容の全てを箇条書き}

## 影響

- {実施した対応内容による影響を箇条書き}

---

{私の入力に対して必要なデータ出力}
```
