# NG → OK 言い換え表（汎用技術用語）

ビジネス職向けに技術用語を言い換えるときの参照表。**ドメイン非依存の一般的な技術用語**のみを扱う。
原則: NG 語はそのまま出さない。OK 列の言い回しに翻訳する。文脈に応じて自然に整える。

## 開発・リリース

| NG（技術用語） | OK（ビジネス職向け） |
|---|---|
| デプロイ / リリース（動詞） | 機能を反映する / 提供を開始する |
| ロールバック | 元の状態に戻す |
| マイグレーション | データの移行作業 |
| ホットフィックス | 緊急の修正 |
| フィーチャーフラグ / トグル | 機能のオン・オフ切り替え（社内設定） |
| ブランチ / マージ | （原則出さない。「修正内容を本番に取り込む」等に集約） |

## 障害・エラー

| NG | OK |
|---|---|
| タイムアウト | 処理が決められた時間内に終わらなかった |
| 500 / 502 / サーバエラー | 画面が正しく表示できない状態 |
| 404 | 対象のページ・データが見つからない状態 |
| 例外 / エラーをスロー | 処理が途中で止まる |
| null / NPE | 必要なデータが入っていなかった |
| バリデーションエラー | 入力内容のチェックで弾かれた |
| デグレ / リグレッション | 以前は動いていた機能が動かなくなった |
| 再現する / リプロ | 同じ操作で同じ問題がもう一度起きる |

## 性能・インフラ

| NG | OK |
|---|---|
| レイテンシ / レスポンスタイム | 画面が表示されるまでの時間 |
| スループット / 負荷 | 一度に処理する量 / アクセスの集中 |
| キャッシュ | 一時的に保存した表示用データ |
| 非同期処理 / バッチ | 裏側でまとめて行う処理（完了まで時間がかかる） |
| リトライ | 自動でやり直す |
| スケール / サーバ増強 | 処理できる量を増やす |
| ダウンタイム | サービスを一時的に利用できない時間 |

## データ・処理

| NG | OK |
|---|---|
| クエリ / SQL | データの検索・集計処理 |
| トランザクション | 一連の処理（途中で失敗したら全部なかったことにする単位） |
| 整合性 / 不整合 | データの食い違い |
| インデックス | 検索を速くするための内部の仕組み |
| 集計 / 再計算 | 数値を計算し直す処理 |
| 同期 / 連携 | 別の場所のデータと内容を合わせる |
| API | 他システムとデータをやり取りする窓口 |

## 言い回しのコツ

- **「なぜ」を業務影響で語る**: 技術的な原因（例: キャッシュの不整合）ではなく、利用者から見た結果（例: 古い数値が表示されていた）で説明する。
- **時間軸を明示する**: 「〜時〜分から〜時〜分まで」のように事実の数値はそのまま運ぶ。
- **断定と未確定を分ける**: 確定した事実は言い切り、調査中のことは「現在調査中です」と明示。両者を混ぜない。
- **専門用語のカッコ書き併記をしない**: 「機能の反映（デプロイ）」のように技術語を残すと、ビジネス職はそこで読みを止める。最初から OK 語だけで完結させる。
