# 不具合の調査状況共有テンプレート

ビジネス職が顧客に経過報告する用。原因が未確定の段階でも共有できる体裁。
暫定回避策がある場合はそれを最上段に近い位置で目立たせる。

```
## [機能名]の不具合について（調査状況）

### 現象
[どの操作で・何が起きるかを、画面と業務の言葉で]

### お客様への影響
[誰が・どの業務で困るか。データが壊れる類かどうかは正直に。影響軽微なら明記]

### 回避策（あれば）
[いま顧客に案内できる回避操作。無ければ「現時点で回避策はありません」と明記]

### 対応状況
- 原因：[特定済み／調査中。調査中なら「調査中」と書く]
- 修正の見込み：[YYYY-MM-DD 目処／未定。確約できないなら「未定」]

### 次回ご報告
[いつ次の続報を出すか。例：「進展があり次第／○日中に改めてご連絡します」]
```

---
### 送付前チェック（エンジニア確認用・社外には出さない）
- [ ] 回避策は実際に有効か（手順に技術用語が残っていないか）
- [ ] 「データが壊れるか」を曖昧にしていないか
- [ ] 修正ETAを確約しすぎていないか（不確実なら「未定」）
- [ ] 原因の技術詳細・内部名が混ざっていないか
