# 顧客通知向けインシデント報告テンプレート

対外（顧客）通知を想定した体裁。ビジネス職が顧客にそのまま共有・転送できるトーン。
**事実が確定している部分だけ**を埋める。未確定は「現在調査中です」と明記する。

```
## ご報告：[事象の一言サマリ]

### 発生事象
[何が起きたかを、お客様の業務・画面の言葉で。技術的な原因は書かない]

### 影響範囲
- 対象：[どのお客様・どのプラン・どの機能か。該当しないものは「影響なし」と明記]
- 期間：[YYYY-MM-DD HH:MM 〜 HH:MM。確定値をそのまま記載]
- 内容：[その間、何ができなかった／どう見えていたか]

### 現在の状況
[復旧済み／対応中。復旧済みなら復旧時刻を明記。対応中なら見込みを正直に]

### 再発防止
[今後の対策。未定なら「対策を検討中です」と書き、約束しすぎない]

### お客様へのお願い（任意）
[再操作・再読み込み・データ確認など、お客様側で必要なアクションがあれば]

ご不便をおかけし申し訳ございません。
```

---
### 送付前チェック（エンジニア確認用・社外には出さない）
- [ ] 発生・復旧の時刻と影響範囲は事実と一致しているか
- [ ] 原因の技術詳細・脆弱性に触れる表現が混ざっていないか
- [ ] 再発防止を確約しすぎていないか（未定は「検討中」に留める）
- [ ] 謝辞・トーンは対外向けとして適切か
