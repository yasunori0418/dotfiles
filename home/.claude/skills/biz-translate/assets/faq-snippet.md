# 「これは仕様です」を角が立たないよう伝える短文テンプレート

不具合ではなく仕様・制約であることを、突き放さずに伝える短文。
「仕様です」で終わらせず、理由と次の一手を添えるのがポイント。

```
[現象]につきましては、[機能名]の現在の動作仕様によるものです。

[なぜそうなっているかを、お客様のメリット・安全側の理由で1文]。

[次の一手：回避方法 / 別機能での代替 / 改善要望として受け付ける旨 のいずれか]
```

## 書き方のコツ

- **「仕様です」で止めない**。理由と代替/次の一手をセットにする。
- **理由は前向きに**。「制限しています」より「誤った操作を防ぐためにこうしています」。
- **要望として受ける余地があるなら明示する**。「ご要望として開発チームに共有します」は、突き放さない締めになる。
- 改善予定が**無い／未定なら約束しない**。「現時点では予定しておりません」と正直に。
