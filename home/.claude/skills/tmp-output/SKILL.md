# ファイル出力先ルール

ユーザーからファイル出力（コード生成・ログ出力・分析結果・調査レポート・ドラフトなど）の依頼を受けた場合のルール。

## 出力先の判断順序

1. **基本**: Claude Codeが開かれているプロジェクトディレクトリのルートに `tmp_claude/` ディレクトリを作成して出力する
2. **優先順位**: ただし、プロジェクト固有のClaude Codeの機能（プロジェクトCLAUDE.md／スキル等）によって出力先が指定された場合は、指定された出力先に出力する
3. 依頼内容や出力される作業内容に基づいて、適切な名前をつけたファイルを出力する

## ファイル名の命名規則

- 説明的で短い名前（例：`nvim_config_analysis.md`、`lua_syntax_errors.txt`）
- 日付やタイムスタンプを含める場合は `YYYYMMDD_` 形式で先頭に付ける
- 拡張子は出力内容に合わせて選択（`.md`、`.txt`、`.json`、`.lua` など）

## 対象外

- 既存ファイルの編集・更新（Editツール対象）
- プロジェクト本体のソースコード作成（プロジェクトの所定位置に出力）
- 外部システム（Notion等）への書き込み（`external-writes` スキル参照）
